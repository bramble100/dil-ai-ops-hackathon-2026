## Domain

AI from a software engineer's perspective. This is not an academic ML research wiki - it's a practitioner's knowledge base focused on **using AI effectively as a tool for building software**.

## Scope

- **Tools & Products:** LLM providers (OpenAI, Anthropic, Google), coding agents (Copilot, Claude Code, Cursor), IDEs, MCP servers, plugins
- **Techniques:** Prompt engineering, RAG, fine-tuning, agent patterns, knowledge management
- **Workflows:** Agent skills, hooks, subagents, custom instructions, automation
- **Patterns:** Code generation, code review, debugging with AI, testing with AI, documentation
- **Ecosystem:** Marketplaces, extensions, MCP protocol, tool integrations
- **Ideas & Opinions:** Thought leadership, emerging patterns, predictions, industry direction

## Categorization Axes

When creating concept pages, consider categorizing along these dimensions:

- **Maturity:** experimental | emerging | established | commodity
- **Audience:** individual developer | team | enterprise
- **Integration depth:** standalone tool | IDE plugin | API | embedded in workflow

## Key Questions This Wiki Should Help Answer

1. What tools/techniques are available for X task?
2. How do different approaches compare (trade-offs, when to use which)?
3. What's emerging that I should pay attention to?
4. What patterns have I found effective in my own work?
5. Where is the field heading?
